# SBI CSV Import Specification — 補助スナップショット

> **正本はNotion PRD §9 を参照してください。**  
> このファイルは実装・テスト時の参照用補助スナップショットです（自動生成対象外）。

---

## 9.1 CSVの形式（実物確認済み）

- 文字コード：**Shift-JIS（cp932）**
- 改行：**LF と NEL（U+0085）が混在**。パース時はNELも改行として扱う。
- 構成：口座・商品区分ごとのセクションが縦に連結された**複数セクション構成**
- 先頭6行：メタ情報（"ポートフォリオ一覧" / "一括表示" / "PTS株価非表示" / "総件数：N件" / "選択範囲：…" / "ページ：N"）
- 各セクション：「セクション見出し行 → 列見出し行 → 明細行（複数） → 小計」
- 末尾に「総合計」セクション
- 米国株・現金残高・取引履歴・配当履歴はこのCSVに含まれない

確認できたセクション：
- 株式（現物/特定預り）
- 株式（現物/NISA預り（成長投資枠））
- 投資信託（金額/特定預り）
- 投資信託（金額/NISA預り（成長投資枠））
- 投資信託（金額/NISA預り（つみたて投資枠））
- 投資信託（金額/旧NISA預り）
- 国内債券（特定預り）

## 9.2 セクション→口座・資産クラスのマッピング

口座と商品種別はセクション見出しから判定する（明細行に口座区分列は無い）。

| セクション見出し | 資産クラス | 口座 |
|----------------|-----------|------|
| 株式（現物/特定預り） | 日本株 | SBI特定 |
| 株式（現物/NISA預り（成長投資枠）） | 日本株 | SBI NISA成長投資枠 |
| 投資信託（金額/特定預り） | 投信 | SBI特定 |
| 投資信託（金額/NISA預り（成長投資枠）） | 投信 | SBI NISA成長投資枠 |
| 投資信託（金額/NISA預り（つみたて投資枠）） | 投信 | SBI NISAつみたて投資枠 |
| 投資信託（金額/旧NISA預り） | 投信 | SBI旧NISA |
| 国内債券（特定預り） | 債券 | SBI特定 |

## 9.3 株式・投資信託の列マッピング

10列構成。すべての値はダブルクォート付き。正の数値には先頭に `＋` が付く。

| CSV列 | Notionプロパティ | 備考 |
|-------|----------------|------|
| 銘柄（コード）/ファンド名 | 外部ID＋資産名 | 株式は最初の空白で分割。投信はコード無し |
| 買付日 | （参考） | `----/--/--` は null扱い |
| 数量 | 数量 | 投信は口数 |
| 取得単価 | 平均取得単価 | 投信は1万口あたり |
| 現在値 | 現在価格 | 投信は1万口あたり基準価額 |
| 前日比 | （参考） | MVPでは未使用 |
| 前日比（％） | （参考） | MVPでは未使用 |
| 損益 | 評価損益 | ＋記号を除去 |
| 損益（％） | 評価損益率 | ＋記号を除去 |
| 評価額 | 評価額 | 投信は小数あり |

取得額の算出（CSVに取得額列は無い）：
- 株式：取得額 = 取得単価 × 数量
- 投信：取得額 = 取得単価 × 数量 ÷ 10000（口数・1万口あたり単価のため）

## 9.4 国内債券の列マッピング

ダブルクォートなし。列構成が株式・投信と異なる。

列：`銘柄, 利率(%), 償還日, 利払日, 買付日, 保有額面, 取得単価, 約定為替, 参考為替, 評価額`

- 銘柄コードは無い。取得単価は額面100あたりの単価。
- 取得額 = 保有額面 × 取得単価 ÷ 100
- MVPでは「評価額」「取得額（算出）」「保有額面」のみHoldingsに反映
- 利率・償還日・利払日・為替はP2

## 9.5 パース・スキップルール

- Shift-JIS（cp932）で読み込み、NEL（0x85バイト）をデコード前にLFに置換
- 先頭メタ6行はスキップ
- 「○○合計」「総合計」見出しと、その直後のヘッダ行・数値行をスキップ
- 列見出し行（「銘柄（コード）」「ファンド名」「銘柄,利率(%)…」で始まる行）はスキップ
- セクション見出し行を検出したら口座・資産クラスを切り替える
- 数値正規化：`＋` 除去、カンマ除去、`--` / `----/--/--` / 空文字 → null
- 株式の先頭セルは最初の空白で「コード」「銘柄名」に分割（`8473`・`186A` 等のグロース市場コードも対応）
- 主キー：`SBI_CSV|口座|外部ID（または資産名）`

## 9.6 エラー処理

- 必須列が無い場合はエラー
- パースできない数値は該当行をスキップしエラー内容に記録
- 全体失敗時はCSV Import DB のステータスを `エラー` に更新
- 部分成功時は作成・更新・スキップ件数を記録

## 9.7 サンプルCSV管理

- 本物CSVはGitに入れない（`.gitignore` 対象）
- 匿名化サンプルは `tests/fixtures/sbi_sample.csv` に保存
- 本物CSVはNotionのSBI CSV Import DBに添付
